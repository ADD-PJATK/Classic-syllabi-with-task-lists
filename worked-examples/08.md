# 1. Goal of the Laboratory

Students will:

* recall why **sorting the entire dataset** per attribute is costly for huge training sets  
* understand **incremental** tree induction with the **Hoeffding bound** (intuition: “enough evidence before splitting”)  
* run a **stream** classifier and compare mindset to batch trees  

---

# 2. Basic Concepts

**Batch decision trees (e.g. CART on all data):**

* Consider many splits; pick the best using **all** rows (or heavy sampling).  

**SLIQ / PRINT (historical ideas):**

* **SLIQ:** pre-sorted attributes + breadth-first growth to reduce repeated sorting.  
* **PRINT:** related line of work on efficient disk-friendly induction (implementation details vary by source).

**VFDT / Hoeffding tree:**

* Rows arrive as a **stream**.  
* For each leaf, track sufficient statistics per attribute.  
* Use **Hoeffding bound**: if the **margin** between the best and second-best split is large enough given **n** examples seen so far, pick that split with high confidence—no need to wait for infinite data.

**Concept drift:** the joint distribution of \(P(y \mid x)\) changes over time; incremental models may need **forgetting** or **windows**.

1. Generate stream: `sklearn.datasets.make_classification` in chunks **or** built-in synthetic stream.

### Path B – No stream library
