# 2. Basic Concepts

**Vector space model:** each document = vector of term weights; similar documents have **small angle** (high cosine similarity).

**TF-IDF:**

* **TF:** term frequency in the document.  
* **IDF:** down-weights terms that appear in **many** documents (common words carry little discriminative signal).  

**Cosine similarity:** \(\cos(\theta) = \frac{\mathbf{d}_1 \cdot \mathbf{d}_2}{\|\mathbf{d}_1\|\|\mathbf{d}_2\|}\) — often preferred for **sparse** text vectors.

**Web text challenges:** boilerplate, duplicates, spam, multilingual content, and **very** long tails of rare pages.

# 3. Building TF-IDF in Python (instructor walkthrough)

### Step 1 – Corpus

Create or load **≥8** short strings (e.g. fake reviews).

### Step 2 – Vectorize

```python
from sklearn.feature_extraction.text import TfidfVectorizer
vectorizer = TfidfVectorizer(stop_words="english", max_features=100)
X = vectorizer.fit_transform(documents)
```

### Step 3 – Cluster

```python
from sklearn.cluster import KMeans
km = KMeans(n_clusters=3, random_state=42, n_init="auto").fit(X)
```

### Step 4 – Inspect

For each cluster, print **indices** of documents and **top terms** (largest mean TF-IDF in that cluster — sklearn sample pattern or manual).
