# 2. Basic Concepts

**Stream clustering:** cannot store all points; maintain **summaries** (micro-clusters) and periodically merge or discard.

**Stream classification:** update model incrementally (`partial_fit` in sklearn-like APIs); evaluate on **incoming** data **before** updating (prequential).

**Why not k-fold on a stream?** No natural “end” of dataset; shuffling breaks temporal order; standard CV assumes a **static** pool.

# 4. Python fallback (instructor demonstration)

```python
# Illustrative only — adapt to your library version
from sklearn.naive_bayes import MultinomialNB
from sklearn.datasets import make_classification

X, y = make_classification(n_samples=5000, n_features=20, random_state=0)
clf = MultinomialNB()
batch = 200
scores = []
for i in range(0, len(X), batch):
    Xb, yb = X[i : i + batch], y[i : i + batch]
    if i > 0:
        scores.append(clf.score(Xb, yb))
    clf.partial_fit(Xb, yb, classes=[0, 1])
```

Discuss: first batch used only for `partial_fit` initialization in some APIs — align with your chosen library.
