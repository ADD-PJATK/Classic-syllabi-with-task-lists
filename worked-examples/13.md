# Exercise List 13

**Course:** Analysis of Large Data Sets (ADD)  
**Laboratory Topic:** Final project – presentation, demo, and report integration  
**Tools:** Team’s stack (RapidMiner / Python / Hadoop components / etc.)

---

# 1. Goal of the Laboratory

Students will:

* deliver a **cohesive demo** of semester work  
* align slides with the **final report** structure  
* complete **individual reflection** paragraphs  

---

# 2. Checklist before the session

### Code / workflow

* [ ] `README` with environment setup (**≤15 min** on a lab machine).  
* [ ] Random **seeds** documented.  
* [ ] **Sample data** in repo; **heavy** data downloaded by script with checksum or URL.  

### Story

* [ ] **One** sentence problem → **one** figure that proves progress → **one** limitation.  

### Academic integrity

* [ ] References for datasets and papers; **no** leaked secrets in Git.  

---

# 3. Example slide deck order (instructor template)

1. Title + team  
2. Problem & stakeholder  
3. Data: source, size, legal/ethical note  
4. Methods (2 slides max—diagram)  
5. Main result (**big** figure)  
6. Baseline comparison table  
7. Demo live **or** 60 s screen recording  
8. Limitations + future work  
9. Q&A backup slide (architecture)  

---

# 4. Report finalization

Cross-check sections against [project.md](../project.md):

* Abstract  
* Individual contributions  
* Reproducibility appendix (optional but encouraged)  

---

# 5. Student Tasks (in-session)

## Task 1 – Dry run

Time the demo; cut optional branches if **> allotted minutes**.

---

## Task 2 – Backup plan

If Wi-Fi fails: **offline** figure PDF + pre-recorded **30 s** clip per team.

---

## Task 3 – Reflection draft

Each student: **three** bullets (hardest part, redo, transferable skill) → expand to **half page** after feedback.

---

# 6. Deliverables

Per course policy:

* **Presentation** (live or as scheduled).  
* **Final team report PDF**.  
* **Individual reflection** file(s) if required.  

See [task list 13](../task-lists/13.md) and [project.md](../project.md) for naming and scope.
