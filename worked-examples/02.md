# 2. Recap and Extensions

**Recap:** Support, confidence, lift; Apriori / FP-Growth.

**Extensions:**

* **Closed itemsets** – itemsets with no proper superset having the same support (reduces redundancy).
* **Maximal itemsets** – itemsets that are not contained in any other frequent itemset.
* **Larger datasets** – working with more transactions and items; tuning minimum support/confidence.

---

# 3. Preparing a Transactional Dataset in RapidMiner

### Step 1 – Data format

Transactional data can be:

* **Transaction ID + item list** (one row per transaction, items in one attribute).
* **Binary matrix** (rows = transactions, columns = items, 0/1).

### Step 2 – Loading in RapidMiner

1. Import CSV (e.g. `transaction_id, items` with items as comma-separated).
2. Use **Nominal to Binomial** or **Split** to get one column per item if required by the operator.
3. Or use **Read Excel/CSV** and then **Numerical to Binomial** for a 0/1 matrix.

### Step 3 – Apply FP-Growth or Apriori

1. Add **FP-Growth** (or **Apriori**) operator.
2. Set **min support** (e.g. 0.1–0.3 for larger datasets).
3. Set **min confidence** (e.g. 0.5–0.7).
4. Run and inspect **frequent itemsets** and **association rules**.

# 5. Student Tasks

## Task 1 – Load a larger transactional dataset

Obtain or create a transactional dataset (e.g. from UCI, Kaggle, or synthetic) with **≥ 50 transactions** and **≥ 5 items**.

Load it into RapidMiner and prepare it for association mining.

Deliverable: Short description of the dataset + screenshot of the prepared data.

---

## Task 2 – Run advanced association mining

1. Run **FP-Growth** (or Apriori) with at least two different **minimum support** values (e.g. 0.15 and 0.25).
2. Compare the number of rules and itemsets.
3. Export the top 10 rules (by lift) for one of the runs.

---

## Task 3 – Business interpretation

Choose **three rules** from your results and for each:

1. State the rule and its support, confidence, and lift.
2. Explain what it means in plain language.
3. Suggest one possible business action (e.g. product placement, discount bundle).
