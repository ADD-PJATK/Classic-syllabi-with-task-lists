# 2. Basic Concepts

**Large-scale association mining:** finding frequent itemsets over billions of baskets usually requires **distributing** work across many machines and **aggregating** counts reliably.

**MapReduce (simplified):**

* **Map:** each worker reads a chunk of input and emits **key–value** pairs, e.g. `(item, 1)` or `(pair, 1)`.  
* **Shuffle:** the system groups all values by key.  
* **Reduce:** for each key, combine values (often **sum**) to produce final counts.

**Randomized / approximate algorithms:** trade exactness for **speed** or **memory** on massive data (syllabus theme); here we only build intuition via deterministic small examples.

# 3. Warm-up: word count as MapReduce

### Step 1 – Input

```
big data big map reduce data map
```

### Step 2 – Map (emit one pair per token)

```
(big,1) (data,1) (big,1) (map,1) (reduce,1) (data,1) (map,1)
```

### Step 3 – Reduce (sum per key)

```
big→2, data→2, map→2, reduce→1
```

# 4. From baskets to counts (instructor demonstration)

**Transactions:**

| ID | Items   |
| -- | ------- |
| T1 | A, B    |
| T2 | A       |
| T3 | B, C    |
| T4 | A, B, C |

### Step 1 – Singleton counts (map-like)

Each transaction emits one count per item in the basket:

* T1 → `(A,1)`, `(B,1)`  
* …

### Step 2 – Reduce

Sum counts: `A:3`, `B:3`, `C:2`, …

### Step 3 – Pair counts (optional extension)

Emit `(min(A,B), max(A,B)), 1` for each pair in each basket; reducer sums → co-occurrence table for **candidate** pairs.
