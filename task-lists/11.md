# Session 11 – Graph mining: PageRank and HITS

*Aligned with syllabus: graph exploration—**PageRank** and **HITS** algorithms; lab: tasks on large graphs.*

---

# Task 1 – Conceptual questions

Answer in your own words:

1. What does **PageRank** measure on a directed web graph?
2. How do **authority** and **hub** scores in **HITS** differ from PageRank scores?
3. Why do both algorithms usually add **teleport** / **random jump** (PageRank) or run iterated normalization (HITS) instead of one matrix multiply?

---

# Task 2 – Pen-and-paper or small script on a toy graph

Consider the directed graph:

* A → B  
* A → C  
* B → C  
* C → B  

1. Draw the graph. Write the **adjacency** structure you use (lists or matrix).
2. Run **two** iterations of **PageRank-style** update **by hand** with damping **d = 0.5** and uniform initialization **1/3** per node—or implement the same in code and paste rounded values after 2 iterations.
3. In **2–3 sentences**, say which node should eventually rank highest and why.

---

# Task 3 – HITS on the same graph

1. Perform **two** alternating HITS updates (normalize so vector max = 1 or L1-normalize—state which).  
2. Which node looks like the strongest **authority** after your two steps?

---

# Task 4 – Real graph snippet (optional)

Run PageRank (library: NetworkX, igraph, etc.) on **≥50** nodes from a public edge list (cite source). Report **top-3** nodes by score.
