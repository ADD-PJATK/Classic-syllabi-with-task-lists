# Session 4 – Dimensionality reduction (SVD, PCA) and EigenFace

---

# Task 1 – Conceptual questions

Answer in your own words:

1. What is **PCA**? What does “principal component” mean?
2. How does **SVD** relate to PCA for a data matrix?
3. What is the **EigenFace** method? Why is PCA useful for face images?
4. What is the difference between **feature selection** and **feature extraction**? Give one example of each.

---

# Task 2 – Face dataset and PCA

Use a **face image dataset** (e.g. Olivetti, LFW subset, or course-provided).

1. Load the images and flatten each to a vector; normalise the values (e.g. to [0, 1] or standardise).
2. Apply **PCA**. Choose the number of components so that **explained variance** is at least **90%** (or use a fixed number, e.g. 20–50). Report the number of components and the cumulative explained variance.
3. Optionally plot **explained variance ratio** (or scree plot) for the first 20–30 components.

---

# Task 3 – Eigenfaces and reconstruction

1. Visualise the **first 3–5 eigenfaces** (principal components reshaped to image dimensions). Include the figures in your report.
2. Select **one face** from the dataset. Reconstruct it using **5, 10, and 20** principal components. Show the original and the three reconstructions.
3. Briefly discuss: how does reconstruction quality change with the number of components?

---

# Task 4 – Short analysis

Answer in a few sentences each:

1. What do the first 1–2 eigenfaces typically capture (e.g. lighting, global shape)?
2. Why might we prefer PCA over keeping all original pixel features for recognition or compression?

---
