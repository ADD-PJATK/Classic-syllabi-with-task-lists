# Session 3 – Recommendation systems

---

# Task 1 – Conceptual questions

Answer in your own words:

1. What is **collaborative filtering**? Explain the difference between **user-based** and **item-based** approaches.
2. What is **content-based recommendation**? How does it differ from collaborative filtering?
3. Why do we need **train/test split** (or cross-validation) when evaluating recommenders?
4. Give **one real-world example** where recommendation systems are used.

---

# Task 2 – Rating dataset and preprocessing

Obtain a **rating dataset** (e.g. MovieLens 100K or similar) with user–item–rating triples.

1. Describe the dataset: number of users, items, ratings; rating scale; sparsity (optional).
2. Split the data into **training** (e.g. 80%) and **test** (20%) sets. Use a fixed seed for reproducibility.
3. Justify briefly why you chose this split (e.g. random, or time-based if applicable).

---

# Task 3 – Build and evaluate a recommender

1. Train a **collaborative filtering** model (user-based or item-based) on the training set. State algorithm and main parameters (e.g. number of neighbors, similarity measure).
2. For the test set, compute **MAE** and **RMSE** (define the formulas and report values).
3. For **2–3 selected users**, list the **top-5 recommended items** they have not rated in the training set. Briefly comment on whether the recommendations seem plausible.

---

# Task 4 – Optional: simple baseline

Implement or use a **simple baseline** (e.g. predict global average rating, or user average). Report its MAE/RMSE on the test set and compare with your collaborative filtering model.

---

# Deliverables

Students must submit:

1. **Concept answers (Task 1)**  
   Clear definitions of collaborative filtering (user vs. item-based), content-based recommendation, and the role of train/test split; one real-world example.

2. **Dataset and split (Task 2)**  
   Dataset description (size, scale, sparsity) and description of the split method and seed.

3. **Model and evaluation (Task 3)**  
   Algorithm and parameters; MAE and RMSE on the test set; top-5 recommendations for 2–3 users with short interpretation.

4. **Baseline (Task 4, optional)**  
   If done: baseline description and comparison of metrics.

PDF report, **max. 4–5 pages**, single file.
